# Plan 08-01: Three-Batch Chained Authority Run

## Sovereign Gate

Pass only if three non-overlapping phone-tokenized HF corpus batches update the adapter checkpoint in sequence on REDMAGIC NX789J, every batch passes RunPod PyTorch comparison, final G1/G3 regressions stay green, and artifacts prove checkpoint chaining and no hidden host data path.

## Tasks

1. Predeclare the objective: `3` batches, `8` sequences per batch, seq128, rank-4 adapter SGD, learning rate `0.01`.
2. On phone, create raw chunk files from the HF corpus for batch offsets `0`, `8`, and `16`; tokenize each chunk into a separate cache with the extended ABI.
3. Run batch 0 from the G5/G6 adapter checkpoint, batch 1 from batch 0's phone checkpoint, and batch 2 from batch 1's phone checkpoint.
4. Pull phone outputs and checkpoint manifests.
5. For each batch, build a RunPod reference using the same phone token cache and the exact phone pre-checkpoint for that batch.
6. Compare token cache parity, layer0/layer1 outputs, adapter gradients, and checkpoint update for all batches.
7. Collect thermal/storage/RSS telemetry.
8. Rerun G1/G3 after the chained run.
9. Emit `gate_result.json`, `falsifier_report.json`, and `08-01-SUMMARY.md`.

## Failure Rules

- Any failed comparator fails the phase.
- Missing checkpoint chain evidence fails the phase.
- Missing thermal/memory/storage telemetry fails the phase.
- Any G1/G3 regression fails the phase.
- A repeated/synthetic hidden-state input fails the phase even if loss decreases.
