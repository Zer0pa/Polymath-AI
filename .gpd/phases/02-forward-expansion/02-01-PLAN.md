# Plan 02-01: Two-Layer Phone OpenCL Forward Stack

## Tasks

- Make the layer runner layer-index-aware without changing tensor semantics.
- Add `--run-opencl-stack PACK0 PACK1 OUT_DIR` so layer 1 consumes the layer 0
  output computed inside the phone process.
- Generate a RunPod PyTorch layer-1 reference pack from the pinned E4B snapshot.
- Build the Android runner on RunPod, push it and the layer-1 pack to REDMAGIC,
  and preserve the previous runner binary before adding the G3 binary.
- Execute the two-layer stack on phone OpenCL.
- Compare final phone layer-1 output against the RunPod PyTorch layer-1
  reference with FP64 per-token cosine over non-pad tokens.
- Record memory high-water telemetry with `getrusage(RUSAGE_SELF).ru_maxrss`.
- Run falsifier review before promoting G3.
